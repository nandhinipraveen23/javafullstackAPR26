E-Commerce Design (Scenario-Based)
🔥 Problem Statement (Real-world constraints)

You are building an e-commerce system with the following rules:

✅ Constraints
Every product must have:
id, name, price, qty
Some products have discount, some don’t
Payment methods should be flexible:
UPI, Credit Card, COD
Delivery charges vary:
Free above ₹1000
Otherwise ₹50
Only online payments get extra 5% discount
You cannot create object of Product directly
Payment logic should be loosely coupled
Add test cases for:
Discount
Payment type
Delivery charge
No discount


 Approach
  Abstract Class → Product
Common fields + partial implementation
Force child classes to define discount (abstract double applyDiscount(double totalAmount))
create :  public double getTotalPrice(), public double getDeliveryCharge(double amount)
 Interface → Payment
Completely abstract → different implementations
